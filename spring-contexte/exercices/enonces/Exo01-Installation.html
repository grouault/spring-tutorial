1 Installation

Importation d'un projet dans Eclipse

Dans le menu Eclipse, sélectionnez File/Import puis Existing Maven Projects

Pointez vers le projet existant Exercice1.

Le projet est en Maven et contient toutes les dépendances nécessaires au projet.

Important : Après toutes modifications faites sur le fichier pom.xml, faites un clic droit sur votre projet puis Maven - Update Project ...

Vérifiez que tout compile (pas de [ERROR] dans la console Maven et un BUILD SUCCESS).

Regardez la classe com.exo.Main pour voir ce qui se passe.

Petit tour des dépendances

Maven gère vos dépendances dans le fichier pom.xml

Pour le moment nous n'avons besoin que de spring-context, mais plus tard nous aurons besoin d'autre artifactId

Dans le cas du Spring il est recommandé de faire usage du dependencyManagement qui permet de fixer les valeurs des versions.

Vous avez aussi une dépendance sur log4J2 et son fichier log4j2.xml

Important : pour un Spring totalement moderne il faudrait faire usage de Spring boot.


2 Première application

Création de la classe Client

Dans le répertoire src créez une classe Java Client.

Faites un clic droit sur src, puis New/class

  • package : com.exo
  • nom : Client
  • superclass : java.lang.Object

Ajoutez un constructeur sans paramètre dans votre classe.

Ajoutez des attributs privés :

  • String nom
  • String prenom
  • int age

Codez/générez les get/set pour chaque propriété (nom, prenom, age).

Surchargez la méthode toString afin d'avoir une vision claire d'une instance (concaténer nom/prenom/age par exemple).

Création du fichier XML pour Spring

Dans le répertoire source src\main\resources\spring éditez le fichier XML mesBeans.xml

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
                           http://www.springframework.org/schema/beans/spring-beans.xsd">

<!-- Declarez vos bean ici -->

</beans>

Déclarez un bean client ayant comme propriétés votre nom, prénom et age

Test

Pour tester notre fichier Spring, nous allons utiliser une classe Java de lancement qui va :

  • Charger le fichier XML Spring via la classe ClassPathXmlApplicationContext
  • Nous donner une instance de notre bean client via la méthode getBean(...)
  • Afficher notre bean client

Dans le répertoire src\main\java\com\exo éditez la classe Java Main.

Ajustez le code afin qu'il récupère votre bean géré par Spring (indiquez son id) :

package com.exo;

// Attention aux import, nous sommes en log4j2
import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;
// Import Spring
import org.springframework.beans.BeansException;
import org.springframework.context.support.ClassPathXmlApplicationContext;


public final class Main {
  private final static Logger LOG = LogManager.getLogger();

  public static void main(String[] args) {
    Main.LOG.debug("-- Debut -- ");
    // Declaration du contexte Spring
    ClassPathXmlApplicationContext appContext = null;
    try {
      // Chargement du fichier Spring
      appContext = new ClassPathXmlApplicationContext("classpath:spring/mesBeans.xml");
      // Récupération de notre instance de client
      Client cl1 = (Client) appContext.getBean("xxxxxxx");
      // Affichage
      Main.LOG.debug(cl1.toString());
    } catch (Exception e) {
      Main.LOG.fatal("Erreur", e);
      System.exit(-1);
    } finally {
      // Quoi qu'il arrive fermeture du contexte Spring
      if (appContext != null) {
        appContext.close();
      }
    }
    Main.LOG.debug("-- Fin -- ");
    System.exit(0);
  }
}

Important : Changez la valeur de la chaîne xxxxxxx en fonction de l'id que vous avez indiqué dans VOTRE fichier Spring.

Plus tard nous ferons des tests unitaires avec Spring, nous n'aurons plus de main.

Lancez l'application et regardez ce qui s'affiche.

En cas d'erreur lisez attentivement les messages Spring, n'hésitez pas à scroller.
