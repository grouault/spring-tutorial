1 Attributs XML

Modification de la classe Client

En continuant à partir de l'exercice 4.

Ajoutez un logueur dans votre classe Client. Dans l'exemple ci-dessous nous ferons usage de Log4J2 :

...
// Attention à vos import
import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;

public class Client {
  private final static Logger LOG = LogManager.getLogger();
  ...
}

Ajoutez une méthode initialiser() et une méthode detruire() dans votre classe Client. Elles seront public, retournent void et ne lèvent pas d'exception.

Ajoutez une trace en niveau info dans les méthodes init, destroy et votre constructeur de Client.

...
	Client.LOG.info("Dans mon Init du client {}", this);
...

Modification du fichier XML Spring

Si vous avez un seul bean client, déclarez en un second.

Indiquez, via les attributs XML (int-method et destroy-method) de votre bean, que vous voulez faire usage des méthodes initialiser et detruire pour votre premier bean client, ne mettez rien sur le second.

Test

Lancez votre classe Main et regardez ce qui s'affiche.

Dans la méthode main, mettez la ligne appContext.close(); en commentaire et relancez votre code pour voir ce qui arrive.

En cas d'erreur lisez attentivement les messages Spring, n'hésitez pas à scroller.

2 Annotations

Modification de la classe Adresse

Ajoutez un logueur dans votre classe Adresse. Nous sommes toujours en Log4J2, la dépendance est déjà en place dans votre fichier Maven pom.xml :

...
// Attention à vos import
import org.apache.logging.log4j.LogManager;
import org.apache.logging.log4j.Logger;

public class Adresse {
  private final static Logger LOG = LogManager.getLogger();
  ...
}

Ajoutez une méthode initialiser() et une méthode detruire() dans votre classe Adresse. Elles seront public, retournent void et ne lèvent pas d'exception.

Ajoutez une trace en niveau info dans les méthodes init, destroy et votre constructeur de Adresse.

Ajoutez sur la méthode initialiser() l'annotation @javax.annotation.PostConstruct.

@PostConstruct
public void initialiser() {
  Adresse.LOG.info("Dans mon Init de Adresse {}", this);
}

Ajoutez sur la méthode detruire() l'annotation @javax.annotation.PreDestroy.

Modification du fichier XML Spring

Si vous avez un seul bean adresse, déclarez en un second.

Indiquez à Spring que vous faites usage des annotations @PreDestroy et @PostConstruct en ajoutant la ligne :.

  <bean class="org.springframework.context.annotation.CommonAnnotationBeanPostProcessor" />

Test

Lancez votre classe Main et regardez ce qui s'affiche.

Remarquez-vous une différence entre l'usage des annotations et l'usage des attributs XML ?.

En cas d'erreur lisez attentivement les messages Spring, n'hésitez pas à scroller.
